DENSITY ist gleichzeitig das Produkt (CLI + Benchmarks + Distribution) und der Algorithmus (konservative, transparente Policy), der Linux-Kernel-Mechanismen wie KSM (Samepage Merging) nutzt – ohne eigene Kernel-Module.

Wofür ist das gut?
  • VM-/Worker-Farms, CI, Test-Labs: viele ähnliche/identische Instanzen.
  • RAM ist teuer: wenn RAM limitiert, gewinnt Dichte direkt Geld & Kapazität.
  • Proof statt Behauptung: Benchmarks + Rohdaten + Methodik.
Sicherheit: In Multi-Tenant-Umgebungen kann Memory-Dedup zusätzliche Side-Channel-Risiken erhöhen. DENSITY ist im MVP primär für trusted / single-tenant Setups gedacht.

Quickstart (MVP)

DENSITY ist (MVP) Linux-orientiert. KSM-Steuerung benötigt Root-Rechte.

Beispiel Linux · sudo 
sudo densityctl enable
densityctl status

# Benchmark + Publish (für GitHub Pages)
sudo densityctl bench --profile P1 --scale 10..80..10 --mem-mib 256 --out results \
  --publish docs/data/benchmarks.latest.json

enable

Aktiviert KSM mit konservativen Defaults. Parameter sind explizit und auditierbar.

status

Zeigt KSM-Stats (pages_shared/pages_sharing etc.) als Text oder JSON.

bench

Reproduzierbare Profile (P1–P3) und Report-Output für Vorher/Nachher.
